# SemanticVersionBuildToolPlugin

A Swift Package Manager build plugin that automatically generates version information for your targets by reading from a `.version` file and creating Swift source code using the `#semanticVersion` macro.

## Overview

The SemanticVersionPlugin is a build tool plugin that:
1. Reads version information from a `.version` file in your package root
2. Generates Swift source code for each target that uses the plugin
3. Creates a `TargetVersion` enum with a static property containing the semantic version

## Features

- **Automatic Version Generation**: Reads from `.version` file and generates Swift code
- **Semantic Version Macro Integration**: Uses the existing `#semanticVersion` macro for compile-time version parsing
- **Per-Target Generation**: Creates version information specific to each target
- **Build System Integration**: Automatically rebuilds when `.version` file changes
- **Xcode Support**: Works with both Swift Package Manager and Xcode projects

## Usage

### 1. Create a `.version` file

Create a `.version` file in your package root containing your semantic version:

```
1.2.3
```

The plugin will trim newlines but won't parse the content - it passes the string directly to the `#semanticVersion` macro.

### 2. Apply the plugin to your target

In your `Package.swift`, add the plugin to your target:

```swift
.target(
    name: "MyTarget",
    dependencies: ["SemanticVersion"],
    plugins: ["SemanticVersionBuildToolPlugin"]
)
```

### 3. Use the generated version

The plugin will generate a Swift file for each target with the following structure:

```swift
// Generated by SemanticVersionBuildToolPlugin
// Do not edit this file manually

import SemanticVersio

/// Version information for the MyTarget target
extension TargetVersions {
  /// The semantic version of this target
  static let myTarget: SematicVersion = .init(1, 2, 3, prereleaseIdentifiers: [], buildMetadataIdentifiers: [])
}
```

### 4. Access the version in your code

```swift
import MyTarget

let version = TargetVersions.myTarget
print("Version: \(version)") // Version: 1.2.3
```

## Generated Code Structure

For a target named `MyTarget`, the plugin generates:

- **File**: `MyTargetVersion.swift` (in build directory)
- **Enum**: `MyTargetVersion`
- **Property**: `static let myTarget` (lowercased target name)
- **Value**: `#semanticVersion("contents_of_.version_file")`

### Target Name Sanitization

The plugin sanitizes target names for Swift compatibility:
- Hyphens (`-`) and spaces (` `) are replaced with underscores (`_`)
- Enum names are capitalized (e.g., `my-target` â†’ `MyTargetVersion`)
- Property names are lowercased (e.g., `my-target` â†’ `mytarget`)

## Examples

### Basic Usage

```swift
// Package.swift
let package = Package(
  name: "MyPackage",
  targets: [
    .target(
      name: "Core",
      dependencies: ["SemanticVersion"],
      plugins: ["SemanticVersionBuildToolPlugin"]
    ),
    .target(
      name: "Networking",
      dependencies: ["SemanticVersion"],
      plugins: ["SemanticVersionBuildToolPlugin"]
    ),
  ]
)
```

With `.version` file containing `2.1.0`, this generates:

```swift
// CoreVersion.swift
extension TargetVersions {
  public static let core = #semanticVersion("2.1.0")
}

// NetworkingVersion.swift  
extension TargetVersions {
  public static let networking = #semanticVersion("2.1.0")
}
```

### Using Version Information

```swift
// In your application code
import Core
import Networking

func printVersions() {
  print("Core version: \(CoreVersion.core)")
  print("Networking version: \(NetworkingVersion.networking)")
  
  // Access semantic version properties
  print("Major version: \(CoreVersion.core.major)")
  print("Is prerelease: \(CoreVersion.core.isPrerelease)")
}
```

### CI/CD Integration

The plugin works well with CI/CD systems where version information is managed externally:

```bash
# Update version in CI
echo "1.2.3-beta.1" > .version

# Build will automatically use new version
swift build
```

## Build Integration

### Swift Package Manager

The plugin automatically integrates with Swift Package Manager:

```bash
# The plugin runs automatically during build
swift build

# Generated files are in the build directory
# .build/plugins/SemanticVersionPlugin/GeneratedSources/
```

### Xcode

For Xcode projects, the plugin also works automatically:

1. Add the plugin to your target in Package.swift
2. Xcode will run the plugin during build
3. Generated files appear in the build directory

## File Locations

- **Input**: `.version` file in package root
- **Output**: `{TargetName}Version.swift` in plugin work directory
- **Build Integration**: Files are automatically included in compilation

## Requirements

- Swift 6.1+
- SemanticVersionMacro dependency (for `#semanticVersion` macro)
- `.version` file in package root

## Troubleshooting

### Plugin Not Running

Ensure your target includes the plugin:

```swift
.target(
    name: "MyTarget",
    plugins: ["SemanticVersionPlugin"]  // Add this
)
```

### Missing SemanticVersionMacro

Add the macro dependency:

```swift
.target(
    name: "MyTarget", 
    dependencies: ["SemanticVersionMacro"],  // Add this
    plugins: ["SemanticVersionPlugin"]
)
```

### Version File Not Found

Ensure `.version` file exists in package root:

```bash
# Check file exists
ls -la .version

# Create if missing
echo "1.0.0" > .version
```

### Build Cache Issues

Clean and rebuild if version changes aren't reflected:

```bash
swift package clean
swift build
```

The plugin passes version strings directly to `#semanticVersion`, which validates at compile time:

```
// .version contains "invalid.version"
// Build will fail with semantic version parsing error
```

### Custom Version Formats

Support for prerelease and build metadata:

```bash
# Prerelease version
echo "1.0.0-alpha.1" > .version

# With build metadata  
echo "1.0.0+build.123" > .version

# Both
echo "1.0.0-beta.2+build.456" > .version
```

## Implementation Details

The plugin consists of two components:

1. **SemanticVersionPlugin**: The build plugin that detects `.version` files and sets up build commands
2. **SemanticVersionGenerator**: The executable tool that reads the version file and generates Swift code

The generated code uses the existing `#semanticVersion` macro, ensuring compile-time validation and optimal performance.
